# 设计说明

## 项目定位

`MADNet Research Agent` 不是单纯的“问答壳子”，而是一个面向遥感视频超分研究场景的 Agent 工作流原型。它的重点不在于一次回答得多华丽，而在于把研究任务拆成可解释、可复核、可迭代的多阶段系统。

## 为什么原始研究流程效率低

遥感 VSR 研究往往存在四个问题：

- 文献来源零散，检索成本高。
- 不同论文的退化假设和实验协议不一致，难以横向比较。
- 数据集、指标、局限性需要人工整理，容易遗漏。
- 最终结论和证据之间缺乏显式映射，难以核查。

## 为什么用多 Agent

如果把这些环节压成一次性生成，问题是：

- 推理路径不可见；
- 中间错误难定位；
- 结论容易混入未经验证的跳跃。

所以这里采用串行多 Agent 结构：

1. `Planner` 负责问题拆解。
2. `Retriever` 负责证据召回。
3. `Analyst` 负责结构化归纳。
4. `Critic` 负责查缺补漏和抑制过度结论。
5. `Writer` 负责将结果输出成报告和推理轨迹。

## 当前工程实现

当前仓库除了最小运行主链外，还补充了：

- `models.py`：状态模型定义；
- `corpus.py`：语料与示例问题加载；
- `prompts.py`：各 Agent 的系统提示词模板；
- `evaluation.py`：证据覆盖度评估；
- `reporting.py`：输出落盘；
- `tests/`：基础单元测试；
- `data/research_questions.json`：示例研究问题集。

## 适合审核的原因

这个项目适合展示给评审，不只是因为“用了 AI”，而是因为它把 Agent 的价值具体化了：

- 有明确场景；
- 有可描述的痛点；
- 有长链推理；
- 有多 Agent 协作；
- 有可追踪输出；
- 有代码与文档相互对应。
